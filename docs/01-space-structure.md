# 페이지 트리 구조

## 최종 권장 구조

Confluence Space 안에서 아래처럼 페이지 트리를 만든다.

```text
Agent 개발현황
├─ W/G
│  ├─ 인력운영
│  ├─ 제도
│  ├─ 임원조직
│  ├─ 노사
│  ├─ 조직문화
│  ├─ 채용
│  ├─ 총무
│  ├─ 집단지성
│  └─ 보상/근태
├─ VD
│  ├─ Agent 과제 페이지들
├─ DA
├─ MX
├─ NW
├─ 의료기기
├─ SR
├─ CDO
└─ 생기연
```

## 핵심 원칙

- `W/G` 아래에만 도메인 페이지를 둔다.
- `VD`, `DA`, `MX` 같은 사업부 아래에는 도메인 페이지를 반복 생성하지 않는다.
- 사업부 과제는 사업부 페이지 바로 아래에 만든다.
- 사업부 과제의 도메인 구분은 `도메인 레이블`과 `페이지 속성`으로 처리한다.

## 왜 이렇게 하느냐

같은 Space 안에서는 같은 제목의 페이지를 반복해서 만들 수 없다.

즉 아래 구조는 한 Space 안에서 충돌할 수 있다.

```text
W/G > 채용
VD > 채용
DA > 채용
```

그래서 트리는 아래처럼 비대칭으로 둔다.

- `W/G`: 도메인 하위 페이지 있음
- `사업부`: 도메인 하위 페이지 없음

그리고 매트릭스 집계는 트리가 아니라 `셀 전용 레이블`로 맞춘다.

## 예시

`W/G` 과제:

```text
W/G
└─ 제도
   ├─ 규정 비교 에이전트
   └─ 제도 문서 요약 에이전트
```

`VD` 과제:

```text
VD
├─ JD 초안 생성 에이전트
└─ 채용 Q&A 에이전트
```

## 루트 페이지 역할

루트 페이지 `Agent 개발현황`은 아래 역할을 한다.

- 첫 페이지 대시보드
- 전체 매트릭스 표시
- 각 개발주체 페이지로 이동하는 진입점

## 제목 규칙

- 개발주체 페이지 제목은 고정값을 쓴다.
  - `W/G`, `VD`, `DA`, `MX`, `NW`, `의료기기`, `SR`, `CDO`, `생기연`
- `W/G` 하위 도메인 페이지 제목도 고정값을 쓴다.
  - `인력운영`, `제도`, `임원조직`, `노사`, `조직문화`, `채용`, `총무`, `집단지성`, `보상/근태`
- 실제 Agent 페이지 제목은 곧 과제명이다.

예:

- `JD 초안 생성 에이전트`
- `규정 비교 에이전트`
- `사내 제안 분류 에이전트`
