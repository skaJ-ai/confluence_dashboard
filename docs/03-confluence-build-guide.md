# Confluence 구축 가이드

이 문서는 Confluence Data Center 9.2.15 기준으로 `Agent 개발현황` 첫 페이지의 단일 매트릭스를 만드는 절차를 정리한 것이다.

## 1. Space 준비

1. 전용 Space를 하나 만든다.
2. 루트 페이지 `Agent 개발현황`을 만든다.
3. 그 아래에 `W/G`, `VD`, `DA`, `MX`, `NW`, `의료기기`, `SR`, `CDO`, `생기연` 페이지를 만든다.
4. `W/G` 아래에만 도메인 페이지 9개를 만든다.

## 2. Agent 과제 페이지 만들기

### W/G 과제

1. 해당 `W/G > 도메인` 페이지 아래에서 새 페이지를 만든다.
2. 페이지 제목은 곧 과제명이므로 짧고 명확하게 쓴다.
3. 페이지에 아래 레이블 4개를 붙인다.
   - `agent-task`
   - `owner-wg`
   - 해당 도메인 레이블 1개
   - 해당 셀 전용 레이블 1개
4. 본문 상단에 `페이지 속성` 매크로를 넣고 공통 컬럼을 채운다.

### 사업부 과제

1. 해당 사업부 페이지(`VD`, `DA` 등) 아래에서 새 페이지를 만든다.
2. 페이지 제목은 곧 과제명이므로 짧고 명확하게 쓴다.
3. 페이지에 아래 레이블 4개를 붙인다.
   - `agent-task`
   - 해당 사업부 레이블 1개
   - 해당 도메인 레이블 1개
   - 해당 셀 전용 레이블 1개
4. 본문 상단에 `페이지 속성` 매크로를 넣고 공통 컬럼을 채운다.

## 4. 각 셀에 페이지 속성 보고서 넣기

기본 설정 원칙은 같다.

| 한글 UI 항목명 | 권장값 |
| --- | --- |
| 레이블 | 해당 `cell-...` 레이블 |
| 공간에서 | `현재 공간` |
| 제목 열 이름 | `과제명` |
| 표시할 열 | `상태` |
| 표시할 항목 수 | `20` |
| 정렬 기준 | `제목` |

핵심:

- 셀 집계는 `레이블`만으로 맞춘다.
- `부모 페이지`는 기본값으로 사용하지 않는다.

## 6. 왜 이렇게 가는가

- 같은 Space 안에서 같은 도메인 페이지 이름을 사업부마다 반복 생성할 수 없다.
- 따라서 트리만으로 `사업부 x 도메인` 교차조건을 표현하기 어렵다.
- 반면 `셀 전용 레이블`은 각 셀을 정확히 1:1로 대표할 수 있다.

즉:

- 트리: 탐색용
- 레이블: 매트릭스 집계용

## 7. HTML은 언제 검토하나

1차 버전은 HTML 없이 간다.

HTML 또는 CSS 우회를 검토할 상황:

- 셀 안에서 제목만 보이게 더 강하게 줄이고 싶을 때
- 매트릭스가 너무 넓어서 시각 표현을 더 제어해야 할 때
- 기본 표 스타일을 크게 바꾸고 싶을 때
