# Space / Page Tree Structure

## 추천 구조

Confluence Space 안에서 아래처럼 페이지 트리를 만든다.

```text
Agent 개발현황
├─ AX&CI Lab
│  ├─ [인력운영] 과제명
│  └─ [채용] 과제명
├─ VD
├─ DA
├─ MX
├─ NW
├─ 의료기기
├─ SR
├─ CDO
└─ 생기연
```

## 왜 이 구조가 좋은가

- 페이지 트리만 봐도 사업부별 소유 구간이 명확하다.
- `상위 페이지` 기준 필터로 사업부별 집계가 쉽다.
- label만으로는 표현이 어려운 문서 소유 관계를 트리로 해결할 수 있다.
- 각 과제 페이지는 독립 문서라 설명, 회의록, 첨부파일을 같이 쌓기 쉽다.

## 루트 페이지 권장 구성

루트 페이지 `Agent 개발현황`에는 아래처럼 바로 현황이 보이게 두는 걸 권장한다.

- 목적
- 운영 원칙
- 전체 개발과제 현황 리포트
- 필요하면 채용 현황, 진행중 과제 같은 추가 리포트
- 사업부 페이지 링크

이번 구조에서는 루트 페이지 자체를 대시보드로 사용한다.

## 사업부 페이지 권장 구성

각 사업부 페이지에는 다음 정도를 둔다.

- 사업부 설명
- 담당자
- 해당 사업부 과제 작성 규칙
- 해당 사업부 과제 목록 링크
- 필요하면 해당 사업부 전용 리포트 매크로

## 과제 페이지 제목 규칙

권장 형식:

```text
[구분] 과제명
```

예시:

- `[채용] JD 초안 생성 에이전트`
- `[보상/근태] 연차 문의 응답 에이전트`
- `[조직문화] 사내 제안 자동 분류 에이전트`

이렇게 해두면 목록에서 봐도 구분이 바로 읽힌다.
