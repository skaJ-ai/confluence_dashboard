# Report Page Template

이 문서는 `Agent 개발현황` 첫 페이지의 매트릭스 셀에 넣을 `페이지 속성 보고서` 설정값 모음이다.

## 1. 페이지 기본 구조

페이지명:

```text
Agent 개발현황
```

페이지 본문에는 수동 표를 먼저 만든다.

권장 헤더:

| 도메인 | W/G | VD | DA | MX | NW | 의료기기 | SR | CDO | 생기연 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |

세로축:

- 인력운영
- 제도
- 임원조직
- 노사
- 조직문화
- 채용
- 총무
- 집단지성
- 보상/근태

## 2. 셀 공통 설정

각 셀에 `페이지 속성 보고서` 매크로를 넣고 아래 값을 기준으로 설정한다.

| 한글 UI 항목명 | 권장값 |
| --- | --- |
| 레이블 | `agent-task` |
| 부모 페이지 | 해당 `개발주체 > 도메인` 페이지 |
| 공간에서 | `현재 공간` |
| 제목 열 이름 | `과제명` |
| 표시할 열 | `상태` |
| 표시할 항목 수 | `20` |
| 정렬 기준 | `제목` |

설명:

- 제목 열에 과제 페이지 제목이 표시된다.
- 페이지 제목을 곧 과제명으로 쓰는 이유가 여기에 있다.
- 셀을 너무 넓히고 싶지 않으면 `표시할 열`은 최소화한다.

## 6. `상위 항목 포함`을 쓰는 경우

기본은 `부모 페이지`를 쓴다.

아래 경우에만 `상위 항목 포함`을 고려한다.

- 도메인 페이지 아래에 중간 정리 페이지를 둘 때
- 과제 페이지가 직계 자식이 아닐 때
- 하위 트리 전체를 한 셀에 묶고 싶을 때

예:

```text
VD
└─ 채용
   ├─ 2026 상반기
   │  └─ JD 초안 생성 에이전트
   └─ 2026 하반기
      └─ 채용 Q&A 에이전트
```

이 경우 `부모 페이지 = VD > 채용`으로는 안 잡히고, `상위 항목 포함 = VD > 채용`이 필요할 수 있다.
