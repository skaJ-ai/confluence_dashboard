# 여기부터 시작

처음 따라할 때는 이 문서만 순서대로 보면 된다.

## 목표

Confluence 첫 페이지 `Agent 개발현황`에 아래 형태의 자동 매트릭스를 만든다.

- 가로축: `W/G, VD, DA, MX, NW, 의료기기, SR, CDO, 생기연`
- 세로축: `인력운영, 제도, 임원조직, 노사, 조직문화, 채용, 총무, 집단지성, 보상/근태`
- 각 셀: 해당 조건에 맞는 Agent 과제명

중요:

- 셀 집계는 `레이블`로 한다.
- `부모 페이지`는 기본값이 아니다.

## 10분 요약

해야 할 일은 5개다.

1. 루트 페이지와 개발주체 페이지를 만든다.
2. `W/G` 아래에만 도메인 페이지를 만든다.
3. 실제 Agent 과제 페이지를 만든다.
4. 과제 페이지에 `페이지 속성`과 `레이블`을 넣는다.
5. 첫 페이지 수동 표 각 셀에서 `cell-...` 레이블로 `페이지 속성 보고서`를 불러온다.

## 가장 짧은 테스트 순서

### 1. 루트 페이지 만든다

페이지명:

```text
Agent 개발현황
```

### 2. 테스트용 사업부 페이지 하나만 만든다

```text
Agent 개발현황
└─ VD
```

### 3. 과제 페이지 하나 만든다

위 `VD` 아래에 페이지를 만든다.

페이지명 예:

```text
JD 초안 생성 에이전트
```

### 4. 레이블 4개 붙인다

```text
agent-task
owner-vd
dm-talent-acquisition
cell-vd-talent-acquisition
```

### 5. 페이지 속성 넣는다

과제 페이지 본문 맨 위에 `페이지 속성` 매크로를 넣고 아래 2열 표를 만든다.

| 항목 | 값 |
| --- | --- |
| 개발주체 | VD |
| 도메인 | 채용 |
| 상태 | 진행중 |
| 담당 | 홍길동 |
| 우선순위 | High |
| 완료예정일 | 2026-04-30 |
| 관련 링크 | Jira-123 |

### 6. 첫 페이지에 테스트용 표 만든다

루트 페이지 `Agent 개발현황`에 아래처럼 아주 작은 표를 만든다.

```text
| 도메인 | VD |
| 채용   | [여기에 보고서 매크로] |
```

### 7. 해당 셀에 페이지 속성 보고서 넣는다

`VD x 채용` 셀 안에 `페이지 속성 보고서` 매크로를 넣고 아래처럼 설정한다.

| 한글 UI 항목명 | 입력값 |
| --- | --- |
| 레이블 | `cell-vd-talent-acquisition` |
| 공간에서 | `현재 공간` |
| 제목 열 이름 | `과제명` |
| 표시할 열 | `상태` |
| 정렬 기준 | `제목` |

이렇게 하면 셀 안에 `JD 초안 생성 에이전트`가 자동으로 보여야 한다.

## 왜 셀 전용 레이블을 쓰는가

Confluence Data Center의 `페이지 속성 보고서`는 레이블 조건을 기준으로 집계하는데, 이 매트릭스에서는 `주체 + 도메인` 교차조건이 필요하다.

그래서 각 셀을 대표하는 전용 레이블을 만든다.

예:

- `cell-wg-policy`
- `cell-vd-policy`
- `cell-gtr-talent-acquisition`

이 방식이면 `수동 표 + 셀별 보고서` 구조가 가장 단순해진다.
