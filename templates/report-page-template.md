# Report Page Template

이 문서는 한글 UI 기준으로 집계 페이지를 만들 때 바로 따라 넣는 설정값 모음이다.

## 1. 루트 페이지 전체 현황

페이지명 예:

```text
Agent 개발현황
```

매크로:

- `페이지 속성 보고서`
- 영문 매크로명: `Content Properties Report` 또는 `Page Properties Report`

설정값:

| 한글 UI 항목명 | 입력값 |
| --- | --- |
| 레이블 | `agent-task` |
| 공간에서 | `현재 공간` |
| 표시할 열 | `사업부, 구분, 과제명, 상태, 담당, 우선순위, 완료예정일` |
| 제목 열 이름 | `과제 페이지` |
| 정렬 기준 | `사업부` |

설명:

- 루트 첫 페이지가 곧 대시보드라면 `상위 항목 포함`은 비워둔다.
- 같은 Space 안에서 이 체계만 운영하면 `레이블 + 현재 공간` 조합으로 충분하다.

## 2. 사업부별 현황

페이지명 예:

```text
VD 개발과제 현황
```

설정값:

| 한글 UI 항목명 | 입력값 |
| --- | --- |
| 레이블 | `agent-task` |
| 상위 항목 포함 | `VD` 페이지 |
| 공간에서 | `현재 공간` |
| 표시할 열 | `구분, 과제명, 상태, 담당, 완료예정일` |
| 제목 열 이름 | `과제 페이지` |
| 정렬 기준 | `구분` |

## 3. 구분별 현황

페이지명 예:

```text
채용 과제 현황
```

설정값:

| 한글 UI 항목명 | 입력값 |
| --- | --- |
| 레이블 | 레이블 필터 2개 사용: `agent-task`, `talent-acquisition` |
| 공간에서 | `현재 공간` |
| 표시할 열 | `사업부, 과제명, 상태, 담당, 완료예정일` |
| 제목 열 이름 | `과제 페이지` |
| 정렬 기준 | `사업부` |

## 4. 사업부 + 구분 조합 현황

예:

```text
VD 채용 과제 현황
```

설정값:

| 한글 UI 항목명 | 입력값 |
| --- | --- |
| 레이블 | 레이블 필터 2개 사용: `agent-task`, `talent-acquisition` |
| 상위 항목 포함 | `VD` 페이지 |
| 공간에서 | `현재 공간` |
| 표시할 열 | `과제명, 상태, 담당, 완료예정일` |
| 정렬 기준 | `완료예정일` |

## 5. 추천 집계 페이지 묶음

- 전체 개발과제 현황
- 사업부별 현황
- 구분별 현황
- 완료예정일 임박 과제
- 진행중 과제

`진행중 과제`처럼 상태 기반 뷰가 더 필요하면, 상태를 label로 추가하기보다 속성 컬럼으로 유지하고 리포트 페이지를 별도로 나누는 쪽이 보통 더 단순하다.
