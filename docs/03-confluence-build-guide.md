# Confluence 구축 가이드

이 문서는 Confluence Data Center 9.2.15 기준으로 `Agent 개발현황` 첫 페이지의 단일 매트릭스를 만드는 절차를 정리한 것이다.

## 1. Space 준비

1. 전용 Space를 하나 만든다.
2. 루트 페이지 `Agent 개발현황`을 만든다.
3. 그 아래에 `W/G`, `VD`, `DA`, `MX`, `NW`, `의료기기`, `SR`, `CDO`, `생기연` 페이지를 만든다.
4. 각 개발주체 페이지 아래에 같은 도메인 페이지 9개를 만든다.

## 2. Agent 과제 페이지 만들기

1. 해당 `개발주체 > 도메인` 페이지 아래에서 새 페이지를 만든다.
2. 페이지 제목은 곧 과제명이므로 짧고 명확하게 쓴다.
3. 페이지에 아래 레이블 3개를 붙인다.
   - `agent-task`
   - 해당 개발주체 레이블 1개
   - 해당 도메인 레이블 1개
4. 본문 상단에 `페이지 속성` 매크로를 넣는다.
5. 매크로 안에 2열 표를 만들고 표 왼쪽 열에 제목 스타일을 적용한다.
6. `templates/agent-task-page-template.md`의 컬럼명대로 값을 채운다.

## 4. 각 셀에 페이지 속성 보고서 넣기

기본 설정 원칙은 같다.

| 한글 UI 항목명 | 권장값 |
| --- | --- |
| 레이블 | `agent-task` |
| 부모 페이지 | 해당 `개발주체 > 도메인` 페이지 |
| 공간에서 | `현재 공간` |
| 제목 열 이름 | `과제명` |
| 표시할 열 | `상태` 또는 최소 컬럼만 |
| 표시할 항목 수 | `20` |
| 정렬 기준 | `제목` 또는 `완료예정일` |

중요:

- 이 구조에서는 `부모 페이지`를 우선 사용한다.
- 과제 페이지가 도메인 페이지의 직계 자식일 때 가장 정확하다.
- 중간에 폴더형 페이지를 더 둘 계획이 있으면 `상위 항목 포함`으로 바꾼다.

## 6. 왜 레이블 + 부모 페이지를 같이 쓰는가

- `부모 페이지`가 구조상 가장 정확한 위치 조건이다.
- `agent-task` 레이블이 있으면 과제 페이지 외의 설명 페이지가 섞이지 않는다.
- 트리와 레이블이 서로 교차 검증 역할을 한다.

## 7. HTML은 언제 검토하나

1차 버전은 HTML 없이 간다.

HTML 또는 CSS 우회를 검토할 상황:

- 셀 안에서 제목만 보이게 더 강하게 줄이고 싶을 때
- 매트릭스가 너무 넓어서 시각 표현을 더 제어해야 할 때
- 기본 표 스타일을 크게 바꾸고 싶을 때

Data Center 9.2.15에서는 HTML 매크로 가능성이 있지만, 관리자 정책과 XSS 위험을 먼저 확인해야 한다.
