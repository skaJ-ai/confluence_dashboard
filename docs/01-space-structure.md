# 페이지 트리 구조

## 최종 권장 구조

Confluence Space 안에서 아래처럼 페이지 트리를 만든다.

```text
Agent 개발현황
├─ W/G
│  ├─ 인력운영
│  ├─ 제도
│  ├─ 임원조직
│  ├─ 노사
│  ├─ 조직문화
│  ├─ 채용
│  ├─ 총무
│  ├─ 집단지성
│  └─ 보상/근태
├─ VD
│  ├─ 인력운영
│  ├─ 제도
│  ├─ 임원조직
│  ├─ 노사
│  ├─ 조직문화
│  ├─ 채용
│  ├─ 총무
│  ├─ 집단지성
│  └─ 보상/근태
├─ DA
├─ MX
├─ NW
├─ 의료기기
├─ SR
├─ CDO
└─ 생기연
```

실제 Agent 과제 페이지는 각 `개발주체 > 도메인` 아래에 직접 넣는다.

## 예시

```text
W/G
└─ 제도
   ├─ 규정 비교 에이전트
   └─ 제도 문서 요약 에이전트
```

```text
VD
└─ 채용
   ├─ JD 초안 생성 에이전트
   └─ 채용 Q&A 에이전트
```

## 왜 이 구조가 좋은가

- 트리만 봐도 `누가 개발했는지`와 `어떤 도메인인지`가 보인다.
- 첫 페이지 매트릭스 셀에서 `부모 페이지` 필터를 쓰기 쉽다.
- 레이블을 여러 개 조합하지 않아도 셀 집계가 가능하다.
- 사용자 입장에서는 경로만 봐도 문서 의미를 이해하기 쉽다.

## 루트 페이지 역할

루트 페이지 `Agent 개발현황`은 아래 역할을 한다.

- 첫 페이지 대시보드
- 전체 매트릭스 표시
- 주요 운영 원칙 안내
- 각 개발주체 페이지로 이동하는 진입점

즉 루트 페이지는 단순 소개 페이지가 아니라 `매트릭스 대시보드`다.

## 개발주체 페이지 역할

각 개발주체 페이지(`W/G`, `VD`, `DA` 등)는 아래 역할을 한다.

- 도메인별 하위 페이지 묶음
- 해당 주체의 문서 네비게이션 허브
- 필요하면 주체 전용 요약 보고서 위치

## 도메인 페이지 역할

각 도메인 페이지(`채용`, `제도`, `조직문화` 등)는 아래 역할을 한다.

- 실제 Agent 과제 페이지의 부모 페이지
- 도메인 설명이나 참고 문서를 두는 위치
- 첫 페이지 매트릭스 셀에서 직접 참조할 기준 페이지

## 제목 규칙

- `개발주체` 페이지 제목은 고정값을 쓴다.
  - `W/G`, `VD`, `DA`, `MX`, `NW`, `의료기기`, `SR`, `CDO`, `생기연`
- `도메인` 페이지 제목도 고정값을 쓴다.
  - `인력운영`, `제도`, `임원조직`, `노사`, `조직문화`, `채용`, `총무`, `집단지성`, `보상/근태`
- 실제 Agent 페이지 제목은 곧 `과제명`이므로 접두사 없이 짧고 명확하게 쓴다.

예:

- `JD 초안 생성 에이전트`
- `규정 비교 에이전트`
- `사내 제안 분류 에이전트`
