# 여기부터 시작

처음 따라할 때는 이 문서만 순서대로 보면 된다.

## 목표

Confluence 첫 페이지 `Agent 개발현황`에 아래 형태의 자동 매트릭스를 만든다.

- 가로축: `W/G, VD, DA, MX, NW, 의료기기, SR, CDO, 생기연`
- 세로축: `인력운영, 제도, 임원조직, 노사, 조직문화, 채용, 총무, 집단지성, 보상/근태`
- 각 셀: 해당 조건에 맞는 Agent 과제명

## 10분 요약

해야 할 일은 5개다.

1. 루트 페이지와 `개발주체` 페이지를 만든다.
2. 각 개발주체 아래에 같은 `도메인` 페이지들을 만든다.
3. 실제 Agent 과제 페이지를 해당 `개발주체 > 도메인` 아래에 만든다.
4. 과제 페이지에 `페이지 속성`과 `레이블`을 넣는다.
5. 첫 페이지에 수동 표를 만들고 각 셀에 `페이지 속성 보고서`를 넣는다.

## 가장 짧은 테스트 순서

### 1. 루트 페이지 만든다

페이지명:

```text
Agent 개발현황
```

### 2. 테스트용 트리 2단계만 만든다

```text
Agent 개발현황
└─ VD
   └─ 채용
```

### 3. 과제 페이지 하나 만든다

위 `VD > 채용` 아래에 페이지를 만든다.

페이지명 예:

```text
JD 초안 생성 에이전트
```

중요:

- 페이지 제목 자체가 나중에 셀에 보이는 `과제명`이 된다.

### 4. 레이블 3개 붙인다

```text
agent-task
owner-vd
dm-talent-acquisition
```

### 5. 페이지 속성 넣는다

과제 페이지 본문 맨 위에 `페이지 속성` 매크로를 넣고 아래 2열 표를 만든다.

| 항목 | 값 |
| --- | --- |
| 개발주체 | VD |
| 도메인 | 채용 |
| 상태 | 진행중 |
| 담당 | 홍길동 |
| 우선순위 | High |
| 완료예정일 | 2026-04-30 |
| 관련 링크 | Jira-123 |

### 6. 첫 페이지에 테스트용 표 만든다

루트 페이지 `Agent 개발현황`에 아래처럼 아주 작은 표를 만든다.

```text
| 도메인 | VD |
| 채용   | [여기에 보고서 매크로] |
```

### 7. 해당 셀에 페이지 속성 보고서 넣는다

`VD x 채용` 셀 안에 `페이지 속성 보고서` 매크로를 넣고 아래처럼 설정한다.

| 한글 UI 항목명 | 입력값 |
| --- | --- |
| 레이블 | `agent-task` |
| 부모 페이지 | `VD > 채용` 페이지 |
| 공간에서 | `현재 공간` |
| 제목 열 이름 | `과제명` |
| 표시할 열 | `상태` |
| 정렬 기준 | `제목` 또는 `완료예정일` |

이렇게 하면 셀 안에 `JD 초안 생성 에이전트`가 자동으로 보여야 한다.

## 왜 `부모 페이지`를 먼저 쓰는가

이 구조에서는 트리가 이미 `개발주체 + 도메인`을 표현한다.

예:

- `VD > 채용`
- `W/G > 채용`

그래서 셀 조건을 잡을 때

- `레이블 = agent-task`
- `부모 페이지 = 해당 도메인 페이지`

조합이 가장 단순하다.

레이블은 보조 식별용으로 유지하면 된다.

## 안 보이면 체크할 것

- 과제 페이지가 정말 `VD > 채용` 아래에 있는지
- 과제 페이지에 `agent-task` 레이블이 붙었는지
- `페이지 속성` 매크로 안의 왼쪽 열에 제목 스타일이 적용됐는지
- 셀 안의 `부모 페이지`가 정확한 `VD > 채용` 페이지로 선택됐는지
- `공간에서`가 `현재 공간`인지
