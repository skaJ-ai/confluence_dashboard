# 레이블과 속성 규칙

## 기본 원칙

- `페이지 트리`는 탐색과 셀 집계 기준이다.
- `레이블`은 검색, 보조 필터, 추가 집계용이다.
- `페이지 속성`은 보고서 표에서 보여줄 값이다.

이 구조에서는 매트릭스 셀 집계를 할 때 `레이블 + 부모 페이지` 조합을 우선 사용한다.

## 필수 규칙

한 Agent 페이지는 반드시 아래를 가진다.

- 공통 레이블 1개
- 개발주체 레이블 1개
- 도메인 레이블 1개
- 페이지 속성 매크로 1개

또한 아래 조건을 반드시 지킨다.

- 개발주체는 정확히 1개
- 도메인은 정확히 1개
- 페이지 제목은 과제명 자체

## 권장 페이지 속성 컬럼

`페이지 속성` 표에는 아래 컬럼을 권장한다.

| 컬럼 | 설명 |
| --- | --- |
| 개발주체 | `W/G`, `VD`, `DA` 등 |
| 도메인 | `채용`, `제도`, `조직문화` 등 |
| 상태 | 아이디어 / 검토중 / 진행중 / 완료 / 보류 |
| 담당 | 실무 담당자 |
| 우선순위 | High / Medium / Low |
| 완료예정일 | 목표 날짜 |
| 목적 | 해결하려는 문제 |
| 관련 링크 | Jira, Drive, Figma 등 |

`과제명`은 페이지 제목으로 관리한다.  
이유는 `페이지 속성 보고서`의 제목 열이 곧 과제명 링크로 보이기 때문이다.

## 왜 레이블을 유지하는가

- 페이지 트리만으로는 전역 검색이 불편하다.
- 나중에 `도메인별 전체 목록`, `개발주체별 전체 목록` 같은 추가 뷰를 만들 수 있다.
- 잘못된 위치에 생성된 페이지도 레이블로 보조 점검이 가능하다.

## 피해야 할 것

- 개발주체 레이블을 두 개 붙이는 것
- 도메인 레이블을 두 개 붙이는 것
- 페이지 제목에 `[채용]` 같은 접두사를 붙이는 것
- 속성 컬럼명을 페이지마다 다르게 쓰는 것
