# 한글 UI 용어 매핑

이 문서는 영문 Atlassian 문서 용어를 한글 UI에서 어떻게 찾아야 하는지 빠르게 보는 용도다.

중요:

- Confluence 버전, 편집기 종류, 사이트 언어 설정에 따라 실제 표기가 조금 다를 수 있다.
- 아래 표는 한글 UI에서 찾기 쉽게 정리한 실무용 매핑이다.
- 문서 본문에서는 한글 용어를 우선 사용하고, 영문은 백업 검색어로만 둔다.

## 매크로 이름

| 한글로 찾을 말 | 영문 문서에서 보일 수 있는 이름 |
| --- | --- |
| 페이지 속성 | `Content Properties`, `Page Properties` |
| 페이지 속성 보고서 | `Content Properties Report`, `Page Properties Report` |

## 페이지 속성 보고서 설정 항목

| 한글 UI에서 찾을 말 | 영문 문서 이름 |
| --- | --- |
| 레이블 | `Labels` |
| 상위 항목 포함 | `With ancestor` |
| 부모 페이지 | `With parent` |
| 공간에서 | `In space` |
| 표시할 열 | `Columns to show` |
| 정렬 기준 | `Sort by` |
| 제목 열 이름 | `Title column heading` |
| 표시할 항목 수 | `Number of items to display` |
| 페이지 속성 ID | `Page Properties ID` 또는 `Content Properties ID` |

## 페이지에서 바로 기억할 것

- `Sort by`를 찾지 말고 `정렬 기준`을 찾는다.
- `Columns to show`를 찾지 말고 `표시할 열`을 찾는다.
- `With ancestor`를 찾지 말고 `상위 항목 포함`을 찾는다.
- `Labels`를 찾지 말고 `레이블`을 찾는다.
- `In space`를 찾지 말고 `공간에서`를 찾는다.

## 지금 질문한 항목 기준으로 바로 답

- `상위 항목 포함`은 `With ancestor`에 해당한다고 보면 된다.
- `부모 페이지`는 `With parent`에 해당한다.
- `공간에서`는 `In space`에 해당한다.
- 과제 페이지와 대시보드 페이지가 모두 같은 Space 안에 있으면 `공간에서 = 현재 공간`으로 두면 된다.
- 그리고 `상위 항목 포함 = Agent 개발현황`을 같이 넣으면, 그 루트 페이지 아래 트리만 집계하게 된다.

## 이 저장소에서의 표기 원칙

- 문서 설명: 한글 UI 기준
- 매크로 검색 백업어: 영어 이름 병기
- 설정 표: `한글 UI 항목명 / 입력값` 형식
